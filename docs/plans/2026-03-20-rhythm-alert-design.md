# 节奏预警系统设计

## 背景

当前 skill 缺乏节奏预警机制。用户写长篇时容易陷入：
- 场景重复（连续多章都在室内对话）
- 伏笔遗忘（埋了10章忘了回收）
- 多线失衡（某条线消失太久）
- 节奏趋同（越写越平）

## 目标

创建节奏预警系统，包含：
1. **脚本** `check_rhythm.py` — 自动扫描章节，生成健康报告
2. **模板** `05-节奏健康报告.md` — 报告格式定义

## 设计方案

### 1. 节奏检查脚本

**文件位置：** `scripts/check_rhythm.py`

**功能：** 扫描小说目录下所有章节，生成节奏健康报告。

**检查项：**

```python
# 场景重复检测
- 检测连续N章是否在同一场景类型（室内/室外/对话/战斗等）
- 标记"场景类型重复超过3章"的情况

# 伏笔回收提醒
- 读取 02-世界观与伏笔.md 中的伏笔列表
- 扫描章节内容，检查哪些伏笔已被回应
- 标记"超过10章未回收"的伏笔

# 章节状态概览
- 字数统计（总字数/平均章节字数）
- 章节密度（每章事件数）
- 钩子强度评估（结尾悬念词检测）

# 多线均衡检查（如存在 03-多线管理.md）
- 读取多线管理中的线状态跟踪表
- 检查各线最后出现章节
- 标记"超过8章未出现"的叙事线
```

**输出示例：**

```text
=== 节奏健康报告 ===
生成时间：2026-03-20
小说：XXX
检查章节：第1-10章

【警告】场景重复
- 室内对话连续出现：第3章、第4章、第5章
- 建议：第6章安排户外场景或动作场景

【警告】伏笔超时未回收
- "神秘信封"：首次出现第2章，至今未回收（已过8章）
- "反派身份"：首次出现第5章，至今未回收（已过5章）

【正常】章节字数
- 平均字数：3200字/章
- 最高：第7章（4100字）
- 最低：第3章（2100字）

【正常】钩子强度
- 有悬念结尾的章节：8/10
- 需要加强：第3章（结尾太平）

【警告】多线均衡（副线A连续消失8章）
- 主线：最近出现第10章
- 副线A：最近出现第2章 ⚠️
- 副线B：最近出现第9章
```

### 2. 节奏健康报告模板

**文件位置：** `novels/<书名>/05-节奏健康报告.md`

**用途：** 每5-10章生成一次报告，记录节奏健康状态。

```markdown
# 节奏健康报告

## 基本信息
- 报告日期：
- 检查范围：第X章 - 第Y章
- 章节数量：N章
- 累计字数：

## 场景分析

### 场景类型分布
| 场景类型 | 出现次数 | 占比 |
|----------|----------|------|
| 室内对话 | 5 | 25% |
| 户外 | 3 | 15% |
| ... | ... | ... |

### 警告
- [ ] 场景类型重复超过3章

## 伏笔跟踪

### 伏笔状态
| 伏笔 | 首次出现 | 最近回收 | 状态 |
|------|----------|----------|------|
| ... | 第X章 | - | ⚠️ 未回收 |

### 警告
- [ ] 有伏笔超过10章未回收

## 字数分析

### 统计
- 总字数：
- 平均字数：
- 最高章节：
- 最低章节：

### 警告
- [ ] 有章节低于2000字
- [ ] 有章节超过5000字

## 钩子强度

### 章节钩子评分
| 章节 | 字数 | 钩子强度 | 备注 |
|------|------|----------|------|
| 第X章 | 3000 | ★★★☆☆ | 结尾太平 |

### 警告
- [ ] 有章节钩子强度低于2星

## 多线均衡（如适用）

### 叙事线状态
| 叙事线 | 最后出现 | 间隔章节 | 状态 |
|--------|----------|----------|------|
| 主线 | 第10章 | 0 | ✅ |
| 副线A | 第2章 | 8 | ⚠️ |

### 警告
- [ ] 有叙事线超过8章未出现

## 综合评估

**健康得分：** ★★★★☆ (4/5)

**主要问题：**
1. 副线A消失太久，需要安排出场
2. 室内对话场景过多

**建议：**
1. 下1-2章优先让副线A出现
2. 增加户外或动作场景平衡
```

### 3. 使用方式

**触发时机：**
- 每写完5-10章后运行一次
- 用户说"检查节奏"时运行

**命令：**
```bash
python3 scripts/check_rhythm.py novels/书名
```
