# 人机协作边界设计

## 背景

当前 skill 是纯 AI 写作流程，但实际使用中，用户往往是"人机混合"写作：
- 用户自己写了某一段，让 AI 继续
- AI 写的初稿，用户修改后让 AI 继续
- 用户提供设定和大纲，AI 负责具体写作

但缺乏明确的协作边界和交接机制，导致：
- AI 写的和用户想的不一样
- 风格不一致
- 上下文丢失

## 目标

创建人机协作系统：
1. **SKILL.md** — 增加 `## 人机协作` 章节
2. **模板** `08-人机协作.md` — 协作分工、风格参考、交接协议

## 设计方案

### 1. 协作分工指南

明确哪些适合人类写，哪些适合 AI 写：

| 内容类型 | 人类写 | AI 写 | 说明 |
|----------|--------|--------|------|
| 核心设定 | ✅ | | 人物性格、核心冲突、底线 |
| 关键情节 | ✅ | | 重大转折、结局设计 |
| 对白风格 | ✅ | | 人物语言特点 |
| 具体场景 | | ✅ | 场景描写、环境渲染 |
| 过渡衔接 | | ✅ | 章节之间的过渡 |
| 扩写 | | ✅ | 已有大纲的详细内容展开 |
| 修改 | ✅ | | 明确指出问题所在 |

### 2. 上下文传递机制

每次让 AI 继续写之前，传递上下文：

```markdown
## 当前进度
- 章节：第X章
- 当前场景：...
- 主要人物：...
- 上文关键事件：...

## 写作要求
- 继续写：...（具体说明要写什么）
- 风格参考：...（文风要求）
- 注意：...（特别注意事项）
```

### 3. 风格参考示例

提供风格示例，让 AI 参照：

```markdown
## 风格参考

### 某角色的对白风格
"..."（示例对白）

### 某场景的描写风格
"..."（示例描写）
```

### 4. 交接协议

人类修改完一段后，告诉 AI：

```markdown
## 人类修改记录
- 修改位置：第X章 第Y段
- 修改原因：...
- 继续要求：...

## 上一段结尾
"..."（引用上段结尾，让 AI 接上）
```
