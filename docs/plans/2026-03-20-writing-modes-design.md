# 写作循环模式分离设计

## 背景

当前的 Chapter Loop 强制每章都走完整 8 步流程，但实际写作有两种截然不同的阶段：

- **草稿阶段**：需要速度，不要打断创作流
- **精修阶段**：需要打磨，逐项检查质量

强制草稿阶段走完整流程会导致：写作被打断、创作热情消退、效率低下。

## 目标

在 SKILL.md 中引入**双模式**写作循环：

1. **草稿模式** — 专注输出，最小打断
2. **精修模式** — 完整检查，质量打磨

让用户在不同阶段使用不同模式。

## 设计方案

### 1. 新增：草稿模式

**定义：** 适用于第一遍快速写作，不打断创作流。

**核心原则：** 写出来比写对更重要。

**步骤（简化版）：**

```
草稿模式步骤：
1. 直接开始写（不读大纲、不拆场景、不查参考文档）
2. 写完后运行字数检查：python3 scripts/check_chapter_wordcount.py <章节文件路径>
3. 标记完成，进入下一章草稿
```

**触发条件：** 用户说"起草第X章"、"先写个初稿"、"快速输出"等。

**判断逻辑：**
- 如果用户明确说"草稿"、"初稿"、"先写"、"快速"，默认草稿模式
- 如果用户说"精修"、"修改"、"打磨"、"质量检查"，默认精修模式
- 如果不确定，默认精修模式（保证质量）

### 2. 精修模式（现有 Chapter Loop）

**定义：** 适用于草稿完成后的质量打磨。

**步骤（完整版）：**

```
精修模式步骤：
1. 读取 00-大纲.md、最近章节摘要、02-世界观与伏笔.md
2. 明确 本章目标 / 阻碍 / 转折 / 结尾钩子
3. 先拆 3-6 个场景，再落正文
4. 开头前 20% 必须尽快进入冲突
5. 对话、扩写、连贯性、结尾钩子分别参考对应文档
6. 写完后运行字数检查
7. 用 quality-checklist.md 自查
8. 回写章节摘要、人物状态、伏笔状态与章节进度
```

### 3. 模式选择指导

在 SKILL.md 的"Default Working Mode"部分之后、Chapter Loop 部分之前，添加：

```markdown
## 写作模式选择

根据当前阶段选择合适的写作模式：

| 阶段 | 推荐模式 | 核心目标 |
|------|----------|----------|
| 第一遍起草 | 草稿模式 | 快速输出，不打断创作流 |
| 修改某一章 | 精修模式 | 逐项检查，打磨质量 |
| 写完所有章节后通读 | 精修模式 | 整体检查前后一致性 |

**判断规则：**
- 用户说"起草"、"初稿"、"先写"、"快速输出" → 草稿模式
- 用户说"修改"、"打磨"、"精修"、"检查质量" → 精修模式
- 不明确时默认精修模式
```
