# 修改工作流设计

## 背景

当前 skill 主要针对"写新章节"，但当用户说"帮我改一下第3章"时，没有系统的修改路径。结果是：改到哪里算哪里，容易漏，容易重复改。

## 目标

在 SKILL.md 中增加**修改工作流**，包含：
1. 问题诊断（先搞清楚哪里有问题）
2. 修改分级（轻度 vs 重度）
3. 针对性修改清单

## 设计方案

### 1. 修改工作流流程

```
修改工作流：

第一步：问题诊断
- 读取目标章节
- 对照质量标准，找出具体问题
- 用一句话描述问题（如"开头300字太拖，没有进冲突"）

第二步：修改分级
根据问题类型和严重程度，判断修改级别：

| 级别 | 问题类型 | 修改范围 | 预计耗时 |
|------|----------|----------|----------|
| 轻度 | 错别字、语句不通顺、标点错误 | 局部修改 | 10-20分钟 |
| 中度 | 某段对话不够自然、某个场景不够具体 | 单场景/单段落 | 30-60分钟 |
| 重度 | 节奏问题、结构问题、开头钩子太弱 | 多场景或全章重写 | 1-3小时 |

第三步：针对性修改
根据诊断结果，按问题类型选择修改策略：
- 钩子太弱 → 参考 hook-techniques.md
- 节奏太慢 → 参考 chapter-guide.md 的节奏部分
- 对白太水 → 参考 dialogue-writing.md
- 场景空洞 → 参考 scene-design.md 和 content-expansion.md

第四步：验证
- 重读修改后的段落，确认问题已解决
- 运行字数检查，确认字数没有严重缩水
```

### 2. 新增章节位置

在 SKILL.md 的 `## 精修模式` 之后、`## Export` 之前，插入新的 `## 修改工作流` 章节。

### 3. 常用问题诊断清单

```markdown
### 常用问题诊断清单

对照以下问题，快速定位章节问题：

**开头问题**
- [ ] 开头是否在20%内进入冲突？
- [ ] 是否有"钩子"吸引读者继续看？
- [ ] 是否有过多的背景/设定说明？

**节奏问题**
- [ ] 是否有过多的"summary"（概述）而非"scene"（场景）？
- [ ] 是否在某段停留太久？
- [ ] 每章是否有明确的事件推进？

**对白问题**
- [ ] 对白是否推进了剧情/关系？
- [ ] 对白是否有多余的"解释腔"？
- [ ] 人物说话是否符合性格？

**结尾问题**
- [ ] 结尾是否有钩子？
- [ ] 是否留下了悬念或转折？

**一致性问题**
- [ ] 是否与前文设定冲突？
- [ ] 人物状态是否连贯？
```
