# 故事引擎独立区块设计

## 背景

当前大纲模板缺少对"故事核心驱动力"的显式表达。作者往往能讲出"这是一个关于xxx的故事"，但说不清"这个故事靠什么机制跑起来"。

结果：写到中段，节奏松散，升级逻辑混乱，主角动机模糊。

## 目标

在大纲模板中增加独立的 `## 故事引擎` 区块，要求作者在开始正文前明确：
- 核心机制是什么
- 代价/资源是什么
- 升级规则是什么

## 设计方案

### 1. 新增区块：`## 故事引擎`

在 `outline-template.md` 中，在"项目定位"之后、"主线冲突"之前插入。

### 2. 子字段定义

```
## 故事引擎

### 核心机制
用一句话说清楚：这个故事靠什么驱动读者往下看？
公式：主角必须___，每___就___。
示例：「主角必须每破解一个暗号才能多活一天，每多用一次能力就失去一段记忆。」

### 代价与资源
每行动一次，主角/世界要失去什么？什么资源是有限的？
示例：「每天只有一次解码机会。用尽后会被强制淘汰出游戏。」

### 升级规则
随着故事推进，代价如何递增？紧张感如何升级？
公式：第N阶段，代价变成___。
示例：「前10章代价是记忆，中10章代价是生命，后10章代价是身边人的命运。」
```

### 3. 修改文件清单

| 文件 | 修改内容 |
|------|----------|
| `references/outline-template.md` | 在"项目定位"后新增完整的故事引擎区块 |
| `SKILL.md` | Intake 部分强调"故事引擎"是必须项；策划模式开始时引导提炼引擎 |
| `references/plot-structures.md` | 增加"如何识别自己的故事引擎"指导 |

### 4. 整合流程

**策划模式开始时，AI 应该：**
1. 先问清题材/类型
2. 再问"这个故事的核心玩点/机制是什么"——用引导问题帮助提炼
3. 引导问题示例：
   - 「读者追这本书的理由是什么？是主角的成长？还是谜题解开的那一刻爽感？」
   - 「主角每赢一次，代价是什么？代价会越来越大吗？」
   - 「这个故事到最后，代价会大到什么程度？」

**验收标准：**
- 故事引擎必须有具体的、可以描述的"机制"
- 不能是空泛的"正义战胜邪恶"或"主角变强"
- 代价必须是读者能感知到的代价，不能是无关痛痒的代价
