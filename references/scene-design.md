# 场景设计指南

章节不是一团连续文字，而是一组有任务的场景。死章通常不是文笔不够，而是场景没有任务、阻碍和结果。

## 场景最小骨架

每个有效场景至少要回答 5 个问题：

1. 谁在推动场景
2. 他这一段想要什么
3. 阻碍是什么
4. 结果是成功、失败还是偏差成功
5. 这段结束后，局面变了什么

如果一段写完后局面没变，这段通常可以删。

## 经典场景引擎

### Scene：推进

适合外部事件推进。

- **Goal**：角色此刻想达成什么
- **Conflict**：谁或什么拦住他
- **Disaster**：结果变差，或付出代价

### Sequel：消化

适合高潮后的反应和下一个决定。

- **Reaction**：人物如何承受刚发生的事
- **Dilemma**：接下来有哪些都不理想的路
- **Decision**：角色做出选择，推进下一场

长篇好看，往往不是全靠 Scene，而是 Scene 和 Sequel 交替得当。

## 章节内场景排序

一个标准强章节，常见顺序如下：

1. 进入点：异常、威胁、选择、误会、坏消息
2. 施压点：阻碍升级，信息不对称
3. 反应点：人物情绪、关系、判断发生变化
4. 转折点：线索反转、关系扭转、计划落空
5. 钩子点：把问题抛到下一章

不要求每章都 5 场，但至少要有进入、施压、转折、钩子。

## 四种常见场景任务

### 1. 推主线

- 线索推进
- 危机升级
- 对抗升级

### 2. 推人物

- 暴露缺陷
- 测试价值观
- 迫使角色做代价性选择

### 3. 推关系

- 信任增加或破裂
- 权力关系变化
- 情感温差变化

### 4. 推设定

- 通过行动暴露规则
- 用代价证明规则
- 在冲突中展示世界观，而不是讲课

## 场景进入原则

尽量晚进早出：

- **晚进**：从已经有张力的位置切入，不从准备动作写起
- **早出**：冲击落下后尽快切走，把余波留给下一场或下一章

### 例子

**弱进入：**
他起床，洗脸，穿衣，出门，准备去见嫌疑人。

**强进入：**
嫌疑人看见他第一眼就跑了。

## 场景里的信息投放

一场里最多优先做 1-2 件核心事：

- 揭示新信息
- 改变关系状态
- 制造新麻烦
- 回收旧伏笔

什么都想做，通常会散；什么都没做，通常会空。

## 死场景信号

- 角色只是在走流程
- 大段解释背景，没有人被影响
- 对话来回很多轮，但关系没变
- 情绪写了很多，决定没有变化
- 细节很多，但没有压力

## 自查清单

- [ ] 这个场景里谁最主动
- [ ] 主动方的目标是否明确
- [ ] 阻碍是否具体，不是笼统“很难”
- [ ] 结尾是否让局面发生变化
- [ ] 下一场是否因此被逼出来
