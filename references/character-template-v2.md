# 人物档案v2（驱动式引擎）

> 人物档案不是静态记录，而是驱动写作决策的引擎。

---

## 核心理念

**v1版本的问题**：填写后变成静态文档，写作时无法调用。

**v2版本的目标**：每个字段都能转化为具体的写作约束。

---

## 核心引擎（必须回答）

### 1. 欲望-恐惧双引擎

**自觉欲望**（主角认为Ta想要的）：
```
例子：成为武林盟主 / 找到真爱 / 查明真相
```

**不自觉欲望**（主角真正需要的）：
```
例子：被认可 / 自我接纳 / 放下执念
```

**核心恐惧**（如果欲望不能实现，最坏的结果）：
```
例子：孤独终老 / 失去自我 / 被所有人背叛
```

**道德前提**（主角相信的"真理"，将在故事中被挑战）：
```
例子："只有强大才能被爱" / "真相总会大白"
```

### 2. 声音指纹

**标志性对白样本**（3-5句，写作时的锚点）：
```markdown
1. "从理论上看，这个问题至少有三个漏洞。"（教授型）
2. "扯淡。那帮人就是想蒙你。"（街头型）
3. "我......我是说，如果你不介意的话......"（害羞型）
```

**禁止用语**（这个人物绝对不会说的话）：
```markdown
- 绝对不会说："我爱你"（只说"你今天看起来不错"）
- 绝对不会说："对不起"（用行动补偿）
- 绝对不会说脏话
```

**情绪状态下的声音变化**：
```markdown
- 愤怒时：短句，重复，打断别人
- 恐惧时：迟疑，破碎句，语速变慢
- 开心时：话多，使用俚语/方言
```

### 3. 缺陷-失败映射

**固有缺陷**：
```
例子：过度自信 / 回避冲突 / 控制欲强
```

**缺陷导致的典型行为**：
```markdown
- 听到反对意见时立刻反驳
- 遇到压力时选择逃避
- 对亲近的人 micromanaging
```

**缺陷可能造成的失败场景**（必须发生2-3次）：
```markdown
1. 第X章：因过度自信低估敌人，导致队友受伤
2. 第Y章：关键决策时因回避冲突失去机会
3. 第Z章：控制欲让重要人物离开
```

**缺陷被克服/被接受的转折点**：
```
第___章：主角意识到缺陷并付出代价，开始改变
```

## 人物弧线（可追踪的里程碑）

### 弧线模板

```markdown
| 章节 | 触发事件 | 变化类型 | 变化描述 | 代价 |
|------|---------|---------|---------|------|
| 第1章 | 初始状态 | - | 懦弱、逃避 | - |
| 第5章 | 好友因Ta的懦弱而受伤 | 前进 | 第一次主动面对 | 好友受伤 |
| 第12章 | 误以为胜利在望 | 倒退 | 再次逃避，失去信任 | 团队解散 |
| 第20章 | 看到逃避的后果 | 前进 | 决心改变 | 承认错误 |
| 第30章 | 最终对决 | 完成 | 勇敢面对，不再逃避 | 重伤但获胜 |
```

### 变化类型定义

- **前进**：向目标状态迈进
- **倒退**：回到旧模式（重要！人物弧线不是直线）
- **完成**：达到目标状态
- **揭示**：隐藏的一面暴露

### 关键规则

1. **必须有倒退**：真实的人物成长是进三步退两步
2. **每次变化必须有代价**：没有代价的改变不真实
3. **变化要可见**：必须体现在行动和选择上

## 关系张力网络

### 双人对立表

对每一对重要关系：

```markdown
### 主角 vs 反派

**欲望冲突**：
- 主角想要：查明真相
- 反派想要：隐藏真相
- 冲突点：只有一个能如愿

**信息差**：
- 主角知道：___
- 反派知道：___
- 观众知道：___（可以高于/低于双方）

**权力关系变化**：
| 章节 | 权力状态 | 主导者 |
|------|---------|--------|
| 第1章 | 反派绝对强势 | 反派 |
| 第10章 | 势均力敌 | 平衡 |
| 第20章 | 主角占据主动 | 主角 |
```

### 群像场景张力

**三人及以上场景的默认张力模式**：

```markdown
### 场景类型：团队决策

**人物A**：想要X（推动）
**人物B**：想要Y（反对A）
**人物C**：想要Z（和事佬/骑墙）

**预设冲突**：
- A和B直接对立
- C的犹豫加剧矛盾
- 决策必须在压力下做出

**场景结果**：
- 有人妥协（关系变化）
- 或者分裂（关系破裂）
```

## 写作调用协议

### 开写前提取（AI自动执行）

```markdown
## 人物约束提取

**主角[XXX]**：
- 核心价值观：___（用于判断行为合理性）
- 当前恐惧：___（用于决策影响）
- 声音锚点：___（3句标志性对白）
- 当前缺陷状态：___（是否准备好付出代价）

**反派[XXX]**：
- 真实动机：___（不是表面目标）
- 与主角的镜像点：___
- 弱点：___（可被利用）

**场景关系状态**：
- A与B：信任度___%（影响互动方式）
- B与C：权力关系___（谁主导）
```

### 写作时实时约束

每生成一段人物相关内容，检查：

- [ ] **行为检查**：这个决策能否由核心价值观+恐惧推导？
- [ ] **对白检查**：是否符合声音锚点？是否使用了禁止用语？
- [ ] **关系检查**：双方互动是否符合当前的信任度/权力关系？
- [ ] **缺陷检查**：缺陷是否影响了决策（正面或负面）？

### 场景结束后更新

```markdown
## 本场景人物变化

**主角[XXX]**：
- 变化：___（情绪/认知/关系/处境）
- 代价：___（付出了什么）
- 下章影响：___

**关系变化**：
- A与B：信任度 ___% → ___%
- 原因：___
```

## 人物一致性自动检查

### 检查项

1. **声音一致性**
   - 本场景对白与声音锚点对比
   - 标记风格突变的句子

2. **行为一致性**
   - 决策是否符合核心价值观
   - 是否与恐惧/欲望冲突

3. **关系一致性**
   - 互动方式是否符合当前关系状态
   - 是否有未铺垫的突然亲近/疏远

4. **弧线一致性**
   - 本场景是否推进了人物弧线
   - 变化是否有代价

### 输出示例

```
人物一致性检查报告：第X章

🔴 声音不一致
位置：第3段
当前："我认为这个假设有三个漏洞。"
问题：太学术，不符合该人物"街头型"声音锚点
建议：改为"扯淡。这说法有三个问题。"

🟡 行为可能有疑问
位置：第10段
行为：主角主动承认错误
问题：与"回避冲突"缺陷冲突
建议：要么增加内心挣扎描写，要么改为被迫承认

✅ 关系一致
A与B的互动符合当前30%信任度设定
```

## 常见错误修正

### ❌ 错误1：完美主角

**表现**：主角聪明、勇敢、善良，没有明显缺陷。

**后果**：
- 缺乏成长空间
- 难以让读者产生共鸣
- 选择没有代价

**修正**：
- 增加一个**致命的**性格缺陷
- 确保缺陷在第5章、第15章、第25章各导致一次失败
- 结局时缺陷被克服或被接受（不是消失，是管理）

### ❌ 错误2：声音同质化

**表现**：所有角色说话都像作者换个名字。

**修正**：
- 为每个主要角色设计3-5句标志性对白
- 设定"禁止用语"（这个人物绝对不会说的话）
- 情绪波动时变化声音特征

### ❌ 错误3：关系跳跃

**表现**：第5章还是陌生人，第6章就成生死之交。

**修正**：
- 使用信任度量化（0%→30%→60%→90%）
- 每次关系变化必须有具体事件支撑
- 倒退同样重要（信任度可以下降）

### ❌ 错误4：弧线直线化

**表现**：主角从懦弱变勇敢是一条直线。

**修正**：
- 设计至少2次"倒退"（以为自己变了，其实没有）
- 每次前进和倒退都有明确触发事件
- 最终克服缺陷时，必须有足够代价铺垫

## 快速启动模板

如果只有10分钟建立人物档案，只填这5项：

```markdown
## 人物快速档案

**名字**：
**欲望**（想要的）：
**恐惧**（最怕失去的）：
**缺陷**（最大的）：
**标志性对白**（3句）：
1.
2.
3.

**禁止用语**（绝对不会说的话）：
```

写作时，对照这5项检查即可。

## 检查清单

每章交付前：

- [ ] 主角行为符合核心价值观
- [ ] 人物对白符合声音锚点（没有使用禁止用语）
- [ ] 关系互动符合当前信任度
- [ ] 本场景是否推进了人物弧线（前进或倒退）
- [ ] 变化是否有代价

---

*好的人物不是被描述出来的，而是在选择中显现出来的。*
