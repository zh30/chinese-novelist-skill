# 首章设计指南

首章不是普通章节。它要一次性完成三件事：

1. 让读者愿意继续读
2. 让读者知道这本书大概会提供什么快感
3. 让主角和核心冲突站住

## 首章必须交付的四样东西

### 1. 人设立住

首章结束前，读者至少要知道：

- 主角是谁
- 主角现在缺什么
- 主角最在意什么
- 主角和别人有什么不一样

### 2. 故事承诺立住

首章要让读者知道这本书主要卖什么：

- 看真相
- 看关系
- 看升级
- 看冒险
- 看群像交汇

如果首章既没立人，也没立承诺，后面再精彩也容易流失。

### 3. 主钩子立住

首章结尾最好抛出一个足够清楚的问题：

- 谁干的
- 为什么会这样
- 主角会不会出事
- 两人会不会靠近
- 这条路到底通向哪里

### 4. 风格立住

首章的语言、密度、节奏，基本决定读者对整本书的预期。不要首章很猛，后面突然变成平铺直叙。

## 首章推荐结构

```text
前 10%：异常 / 危机 / 反差切入
前 30%：主角出场并展现鲜明反应
中段：局势升级，抛出更大的问题
后段：角色被迫做出选择
结尾：主钩子落下
```

## 首章最容易犯的错

### 1. 设定倾倒

一上来讲世界观、历史、门派、家谱、系统说明，读者还没在意人，就先被信息压垮。

### 2. 主角太被动

主角一直在看、听、接受，没有判断和选择，读者很难记住他。

### 3. 冲突太晚

写了很多铺垫，前三段还看不到异常、压力、目标或危险。

### 4. 钩子太虚

首章结尾只是“他陷入沉思”“她看向远方”，没有具体问题，追读率会很差。

## 不同题材的首章重点

### 悬疑 / 推理

- 尽快给案件或异常
- 主角最好在处理问题时出场
- 线索要少而准，不要一章塞满

### 言情

- 尽快让关系对象出现，哪怕只短暂露面
- 先给吸引，再给阻碍
- 情绪要具体，不要空喊悸动

### 奇幻 / 仙侠 / 科幻

- 先给异常和代价，再慢慢解释规则
- 新名词不宜密集轰炸
- 首章最好有一次能体现世界独特性的事件

### 都市现实

- 先给现实压力和人物处境
- 细节要具体，别悬浮
- 开头事件要能照见主角的人生困境

## 首章自查

- [ ] 前 3 段是否已经出现张力
- [ ] 主角有没有做出判断或选择
- [ ] 读者能否说出这本书主要卖点
- [ ] 结尾是否留下具体而非空泛的问题
- [ ] 如果只读首章，读者会不会想点下一章
