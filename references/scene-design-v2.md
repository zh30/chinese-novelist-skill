# 场景设计v2（可执行工具版）

> 从"概念理解"到"可执行工具"，降低场景设计门槛。

---

## 场景核心三问

每个场景开始前，必须回答：

1. **谁最想要某样东西？**（主动方）
2. **什么在阻止Ta？**（阻碍）
3. **最后谁赢了？**（结果）

如果答不上来，这个场景可能没有存在的必要。

## 场景任务检查卡

### 四种任务类型（量化标准）

#### 推主线任务

**完成标准（满足1项即可）**：
- [ ] 揭示了新线索/信息
- [ ] 危机升级了
- [ ] 对抗双方首次正面交锋

**失败信号**：
- 场景结束后，主线没有前进
- 只是"聊了聊"，没有新信息

#### 推人物任务

**完成标准（满足1项即可）**：
- [ ] 角色做出了不符合以往形象的选择
- [ ] 角色的某个秘密被揭示
- [ ] 角色被迫说出"我绝不会..."后立刻打破誓言

**失败信号**：
- 角色的反应完全符合读者预期
- 情绪变化只停留在描写层面，没有转化为行动

#### 推关系任务

**完成标准（满足1项即可）**：
- [ ] 信任度发生了可感知的变化（增加或减少）
- [ ] 权力关系发生了转移
- [ ] 双方说出了"不能收回的话"

**失败信号**：
- 对话很多，但关系状态没变
- 互动结束后，双方立场和之前完全一样

#### 推设定任务

**完成标准（满足1项即可）**：
- [ ] 通过行动暴露了规则（不是讲课）
- [ ] 用代价证明了规则（角色因规则受伤）
- [ ] 在冲突中展示了世界观（角色被迫适应规则）

**失败信号**：
- 大段背景说明，没有角色被影响
- "这是因为..."的解释性叙述

## Scene/Sequel 实用指南

### 什么时候用 Scene（推进）？

**特征**：
- 外部事件为主
- 目标明确
- 有明确输赢

**典型场景**：
- 战斗、追逐、谈判、比赛
- 调查取证、寻找线索
- 逃脱、潜入、拦截

### 什么时候用 Sequel（消化）？

**特征**：
- 内部反应为主
- 没有明确输赢
- 为下一个决定做准备

**典型场景**：
- 失败后的情绪处理
- 新信息后的重新评估
- 重大选择前的犹豫

### 节奏配比表

| 章节类型 | Scene 数量 | Sequel 数量 | 推荐模式 |
|---------|-----------|------------|---------|
| 强动作章 | 3-4 | 0-1 | Scene-Scene-Scene-Sequel |
| 情感转折章 | 1-2 | 2-3 | Scene-Sequel-Sequel-Scene |
| 信息揭示章 | 2 | 1-2 | Scene-Sequel-Scene |
| 多线并行章 | 2-3 | 0 | 纯 Scene 快速切换 |
| 喘息章 | 1 | 2-3 | Sequel-Scene-Sequel |

### Sequel 长度判断

**刚才有大事件改变人物命运** → Sequel 可以长（500-800字）
- 示例：主角刚失去重要人物

**只是日常推进** → Sequel 应该短（200字以内）
- 示例：完成一次普通调查

**连续多个 Scene 后** → 必须有一个较长的 Sequel
- 让读者喘息，让角色消化

## 晚进早出方法论

### 不同类型场景的切入点评分

#### 动作场景

| 切入点类型 | 得分 | 示例 |
|-----------|------|------|
| 直接进入冲突高潮（已经在打/逃/追） | +2 | "子弹擦着他的耳朵飞过。" |
| 从异常发现开始（发现尸体/密室/痕迹） | +1 | "她推开门，发现地上有一滩血。" |
| 从准备行动开始（决定去打/正在准备装备） | 0 | "他决定去教训那个人。" |
| 从日常开始（起床/吃饭/聊天） | -1 | "他起床，洗漱，准备出门。" |

**判断标准**：
- ≥+1分：可用
- ≥+2分：推荐
- ≤0分：建议重写

#### 对话场景

| 切入点类型 | 得分 | 示例 |
|-----------|------|------|
| 从冲突中间切入（已经在争吵/对峙） | +2 | "你骗我！"她摔门而去。" |
| 从关键信息开始（一句话改变一切） | +1 | "你父亲不是意外死的。" |
| 从寒暄开始（铺垫太长） | 0 | "最近怎么样？" "还不错。" |
| 从背景介绍开始 | -1 | "我们认识三年了，从大学开始..." |

#### 内心独白场景

| 切入点类型 | 得分 | 示例 |
|-----------|------|------|
| 从情绪峰值切入（已经在崩溃/顿悟） | +2 | "那一刻，她终于明白了一切。" |
| 从触发事件开始（某事引发了思考） | +1 | "看到那封信，她开始怀疑。" |
| 从平静状态开始（慢慢进入） | 0 | "她坐在窗前，想着最近的事。" |
| 从背景回忆开始 | -1 | "她想起了三年前的那一天..." |

### 早出原则

**冲击落下后，最多再写2段就应该切走。**

**弱收尾**：
```
他扣动扳机，敌人倒下了。然后他走上前，检查了伤口，
确认对方已经死了。他长出一口气，把枪收好，整理了一下
衣服，转身离开现场。走出大门时，他回头看了一眼...
```

**强收尾**：
```
他扣动扳机。

敌人倒下。

他转身离开，没有回头。
```

## 六种场景过渡技术

### 1. 因果链过渡（最常用）

上一场的结果 = 下一场的起因

```
A场景：主角发现被跟踪
↓
B场景：主角开始逃跑/设反跟踪
```

**适用**：连续动作、追踪、逃脱

### 2. 悬念转移

把上一场的悬念带到下一场开头

```
A场景结尾："门开了"（悬念：谁来了？）
↓
B场景开头："门外站着的是..."（揭晓）
```

**适用**：悬疑、揭示、意外

### 3. 情绪对比

上一场的高潮情绪 vs 下一场的平静/反差

```
A场景：激烈争吵（高潮）
↓
B场景：独自沉默整理情绪（低谷）
```

**适用**：情绪调节、深度刻画

### 4. 时间压缩

用一句话跳过中间时间

```
"三天后..."
"直到午夜，他才..."
"一周后，当一切平息..."
```

**适用**：过渡时期、非关键时间

### 5. 空间跳转

明确的空间转移信号

```
"与此同时，在城市的另一端..."
"千里之外的总部..."
"与此同时..."
```

**适用**：多线叙事、并行事件

### 6. POV 切换

切换到另一个角色的视角

```
A场景：主角发现线索（主角视角）
↓
B场景：反派得知计划泄露（反派视角）
```

**适用**：多视角小说、展示信息差

## 场景价值测试（死场景识别）

回答以下5个问题：

1. **如果删除这个场景，读者会看不懂后续剧情吗？**
   - [ ] 会 → 保留，但需要检查是否更高效
   - [ ] 不会 → 进入问题2

2. **这个场景是否改变了至少一个角色的状态？**
   - [ ] 是（情绪/认知/关系/处境）→ 保留
   - [ ] 否 → 进入问题3

3. **这个场景是否揭示了让读者惊讶的新信息？**
   - [ ] 是 → 保留，但检查能否并入其他场景
   - [ ] 否 → 进入问题4

4. **这个场景的对话是否"不能发生在其他场景"？**
   - [ ] 是 → 保留（独特情境）
   - [ ] 否 → 考虑删除或重写

5. **这个场景的氛围/情绪是否为后续必需的铺垫？**
   - [ ] 是 → 缩短到最小有效长度
   - [ ] 否 → 删除

**测试结果**：
- 第1题选"会" → 保留（核心场景）
- 第2-3题任选一"是" → 保留（功能场景）
- 全部"否" → 考虑删除

## 情绪曲线设计模板

### 单场景内情绪变化

```markdown
## 场景情绪曲线

**起点情绪**：___（如：平静/紧张/愤怒）
**情绪触发点**：___（具体事件，200字内必须出现）
**情绪峰值**：___（最强烈的情绪，场景中段）
**情绪落点**：___（场景结束时的情绪）

**检查**：
- [ ] 情绪是否有明确变化？（没有变化 = 死场景）
- [ ] 情绪变化是否驱动了行动？（情绪→决策→行动）
- [ ] 情绪落点是否自然引出下一场？
```

### 情绪曲线示例

```
场景：主角发现被背叛

起点：信任（90%）
      ↓
触发：看到证据（第3段）
      ↓
峰值：愤怒+痛苦（第8段）
      ↓
落点：决心报复（第12段）→ 引出下一场行动

情绪变化：信任 → 怀疑 → 愤怒 → 决心
```

## 功能性场景处理指南

### 什么是功能性场景

"必须存在但不好看"的场景：
- 解释世界观规则
- 人物背景介绍
- 过渡性情节

### 处理策略

#### 策略1：合并法

把功能信息并入其他有冲突的场景

```
不要：专门开一场解释修炼体系
要：在生死战斗中，主角被迫使用规则取胜

不要：回忆主角童年
要：反派揭露主角不知道的身世秘密
```

#### 策略2：伪装法

用有张力的形式包装功能信息

```
不要：主角自言自语介绍世界
要：外来者提问，本地人回答（信息不对称产生张力）

不要：旁白解释历史
要：两人在争论历史真相（观点冲突产生张力）
```

#### 策略3：加速法

功能场景快速完成，把篇幅留给精彩场景

```
字数标准：功能场景 ≤ 平均场景的30%

示例：
过渡场景：500字（交代必须信息）
平均场景：2000字
精彩场景：3000字
```

#### 策略4：跳笔法

完全跳过，用叙述代替场景

```
"三天后，他们抵达了王城。期间一切顺利。"

"一周后的会议上，所有人都同意了这个方案。"
```

**适用**：
- 旅程（除非途中有事件）
- 准备过程（除非遇到阻碍）
- 时间流逝（除非期间有变化）

## 多POV场景切换指南

### 切换时机

**必须切换时**：
- 需要展示主角不知道的信息
- 需要展示反派的计划
- 需要展示并行发生的事件

**避免频繁切换**：
- 同一场景内多次切换（让读者混乱）
- 每章都切换（失去主线连贯性）
- 切换到不重要的角色（分散焦点）

### 切换标记

```
#### 场景A：主角视角
...

---

#### 场景B：反派视角（3小时后）
...

---

#### 场景C：主角视角（继续）
```

### 切换后快速定位

每次切换后，用1句话快速建立新视角：

```
❌ 错误：
他看着窗外。他不知道的是，她正在做准备。

✅ 正确：
他看着窗外。

---

她正在整理装备——三把刀、两包炸药，和一张地图。
（新视角，直接展示行动）
```

## 场景自查清单

每章交付前：

- [ ] 每个场景有明确任务（推主线/人物/关系/设定）
- [ ] 每个场景的切入点评分 ≥ +1
- [ ] 场景之间有过渡（不是生硬跳跃）
- [ ] 没有连续3个Scene没有Sequel
- [ ] 每个场景有价值（通过价值测试至少1项）
- [ ] 功能性场景已合并/伪装/加速/跳笔

## 常见错误修正

### ❌ 场景没有任务

**表现**：角色在聊天、吃饭、走路，什么都没发生。

**修正**：
- 给场景一个明确的任务
- 或者删掉，用叙述代替

### ❌ 场景切入太晚

**表现**：铺垫太长，读者已经失去兴趣。

**修正**：
- 使用切入点评分表
- 从冲突/异常开始，不要从日常开始

### ❌ 场景结束太晚

**表现**：高潮后还写了很多"余波"。

**修正**：
- 高潮后最多2段，然后切走
- 余波留给下一场或叙述带过

### ❌ 场景之间没有过渡

**表现**：场景A结束，突然跳到场景B，读者困惑。

**修正**：
- 使用六种过渡技术之一
- 至少让读者知道"时间/地点/视角"变了

### ❌ 功能性场景太长

**表现**：大段世界观解释、背景介绍。

**修正**：
- 使用四种处理策略之一
- 字数控制在平均场景的30%以内
