# 人机协作协议v2（快速交接版）

> 从"正式化交接"到"一句话续写"，降低协作门槛。

---

## 核心理念

**v1的问题**：上下文传递格式太正式，用户需要手动填写很多内容。

**v2的目标**：AI自动提取关键信息，用户只需说"继续写"。

---

## 快速续写模式（默认）

### 用户说... AI自动做...

| 用户输入 | AI动作 |
|---------|--------|
| "继续写" / "下一章" | 读取进度仪表盘 → 提取上下文 → 自动场景拆分 → 开始写作 |
| "写第X章" | 读取第X章任务卡 → 提取关键上下文 → 开始写作 |
| "从[某处]继续" | 定位到该位置 → 读取前后文 → 续写 |
| "修改第X章" | 读取该章 → 诊断问题 → 给出修改方案 |

### AI自动提取的信息

当用户说"继续写"时，AI自动从以下位置提取：

1. **进度仪表盘**（`99-进度仪表盘.md`）
   - 当前写到第几章
   - 本章状态
   - 下章任务卡

2. **悬念追踪表**（`03-悬念追踪表.md`）
   - 活跃悬念（待推进）
   - 即将过期悬念（需提及）
   - 悬念过期警告

3. **人物档案v2**（`01-人物档案-v2.md`）
   - 主角：核心价值观、当前恐惧、声音锚点
   - 反派：真实动机、弱点
   - 关系状态：信任度、权力关系

4. **上一章正文**
   - 最后500字（或最后一个场景）
   - 结尾钩子

5. **当前章任务卡**（如已存在）
   - 本章目标
   - 场景拆分

**提取时间**：< 2秒

**提取后自动输出**：
```
已读取上下文：
- 当前：第X章（场景拆分完成，开始正文）
- 上章结尾：主角发现...
- 活跃悬念：3个（001凶手身份、002主角身世...）
- 人物状态：主角在[位置]，情绪[状态]
- 本章目标：[目标]

开始写作...
```

## 信息丢失防护

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| AI忘记之前的伏笔 | 上下文太长被截断 | 使用悬念追踪表，AI自动读取 |
| 人物性格突变 | 没有声音锚点 | 人物档案v2包含3句标志性对白 |
| 关系状态错误 | 没有关系量化 | 进度仪表盘显示当前信任度 |
| 悬念被遗漏 | 没有过期提醒 | 自动检测5章/10章未提及的悬念 |

### 防丢失机制

1. **关键信息冗余存储**
   - 重要设定同时存在于：大纲、人物档案、进度仪表盘
   - AI写作前从多个来源交叉验证

2. **自动检测不一致**
   - 人物行为与核心价值观冲突时警告
   - 对话与声音锚点不符时提示
   - 关系变化没有事件支撑时询问

3. **版本回滚**
   - 每章保存历史版本
   - 可查看"人物在第X章时的状态"

## 多轮对话简化

### v1版（太繁琐）

```
人类：继续写
AI：好的，我需要先读取以下文件：
1. 00-大纲.md
2. 01-人物档案.md
3. 02-世界观与伏笔.md
4. 上一章正文
请确认...

人类：确认
AI：正在读取...
（等待）

AI：已读取。当前人物状态是___，悬念状态是___。
我现在开始写第X章第Y场景。场景拆分如下：
1. ...
2. ...
请确认后开始写作。

人类：确认
AI：开始写作...
```

### v2版（一句话搞定）

```
人类：继续写
AI：已读取（第X章，3个活跃悬念，人物在[位置]）。
正在写第X章第Y场景...

（直接开始写作）
```

**变化**：
- 自动读取，无需确认
- 自动场景拆分，无需提前展示
- 关键信息简化为一句话摘要
- 有问题时才询问

## 特殊情况处理

### 场景1：AI写的和用户预期不符

**用户**："不对，人物不应该这样反应"

**AI**：
```
了解。根据人物档案v2，[角色]的核心价值观是___，当前恐惧是___。
您希望的反应是___？
选项：
1. 保持设定，重写此段
2. 调整人物设定（是否更新档案？）
3. 这是特殊情况，人物会打破常规（需铺垫）
```

### 场景2：用户想跳过某些步骤

**用户**："直接写正文，不要场景拆分"

**AI**：
```
进入快速模式。跳过场景拆分，直接写作。
关键信息：第X章，目标是___，结尾需要有___钩子。
开始写作...
```

### 场景3：长时间中断后恢复

**用户**："继续写"（距离上次已过去N天）

**AI**：
```
欢迎回来。已读取进度：
- 上次写到：第X章第Y场景
- 距今天数：N天
- 关键提醒：悬念[002]已5章未提及，建议本章提及

快速回顾：
上章结尾：___
当前任务：___

开始写作...
```

## 协议降级（紧急情况）

如果文件系统损坏或信息缺失，使用极简协议：

```markdown
## 极简续写协议

**必须提供**（用户口述）：
1. 当前写到第几章？
2. 上一章结尾发生了什么？（50字）
3. 主角现在在哪？要干什么？
4. 有没有特别需要注意的？

**AI动作**：
基于以上4点，开始写作。
```

## 检查清单

每次交接时自动检查：

### 从AI到人类
- [ ] 悬念推进/提及状态已更新
- [ ] 人物状态变化已记录
- [ ] 下章任务卡已生成（或提醒）
- [ ] 字数已统计
- [ ] 质量检查已完成

### 从人类到AI
- [ ] 关键修改已说明
- [ ] 继续要求已明确
- [ ] 上下文引用已提供（最后50字）
