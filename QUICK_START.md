# 快速上手指记

> 5 分钟学会使用 chinese-novelist-skill 写小说

---

## Step 0: 安装（1 分钟）

### 如果你用 Claude Code

```bash
# 克隆到 skill 目录
git clone https://github.com/henry/chinese-novelist-skill.git ~/.claude/skills/chinese-novelist-skill

# 重启 Claude Code
# 或者重新加载 skill（具体命令取决于你的工具）
```

### 如果你用其他工具

参考你的工具文档，将本仓库放到 skill 目录即可。

---

## Step 1: 创建第一本小说（5 分钟）

### 对 AI 说

```text
使用 chinese-novelist-skill，帮我写一本悬疑小说。
```

### AI 会引导你

1. **填写极简大纲**（10 个问题，5 分钟）
   - 主角是谁，想要什么？
   - 核心冲突是什么？
   - 结局是什么？
   - 分几章，每章干嘛？
   - ...（共 10 个）

2. **自动生成文件**
   - `novels/我的悬疑小说/00-大纲.md`
   - `novels/我的悬疑小说/99-进度仪表盘.md`

3. **开始写第 1 章**
   - AI 会自动：
     - 读取大纲
     - 设计首章钩子
     - 写正文

### 你只需做

- [ ] 回答 10 个极简问题
- [ ] 确认书名（AI 会生成 3-5 个候选）
- [ ] 等待 AI 写完第 1 章

---

## Step 2: 继续写作（1 分钟）

### 对 AI 说

```text
继续写
```

### AI 会自动

1. 读取 `99-进度仪表盘.md` 获取当前状态
2. 检查悬念过期警告
3. 提取人物当前状态
4. 设计下一章钩子
5. 写正文

### 你只需做

- [ ] 阅读 AI 写的章节
- [ ] 如需修改，说"修改第 X 章，问题在..."
- [ ] 如满意，说"继续写"写下一章

---

## Step 3: 质量检查（2 分钟）

### 每章完成后运行

```bash
# 检查字数
python scripts/check_chapter_wordcount.py novels/我的悬疑小说/第01章.md

# 检查 AI味
python scripts/check_ai_style.py novels/我的悬疑小说/第01章.md
```

### AI 味检测报告示例

```
🤖 AI味检测报告：第01章

🔴 AI味评级：中度（综合得分: 4.2）

❌ 空泛形容词: 5/千字 (阈值: <3) 🔴 超标
   示例: "他很悲伤" "她非常开心"
   💡 建议: 用具体动作替代情绪标签

📚 参考文档: references/ai-style-examples.md
```

### 修复 AI 味

打开 `references/ai-style-examples.md`，找到对应症状的改写范例：

| 原句 | 改写 |
|------|------|
| 他很悲伤 | 他盯着手机屏幕，拇指在玻璃上滑了三次 |
| 她感到前所未有的喜悦 | 她差点撞上门框，手里的咖啡洒了一半也没在意 |

修改后保存，再次运行检测。

## Step 4: 管理悬念（进阶）

### 查看悬念状态

在 `99-进度仪表盘.md` 中查看：

```markdown
## ⚠️ 关键提醒

### 🔔 悬念状态
| 悬念 ID | 内容 | 状态 |
|--------|------|------|
| 001 | 凶手是谁 | 🟢 活跃（最近第 8 章提及） |
| 002 | 主角身世 | 🔴 已过期（已 10 章未提及） |

**本章建议**：提及悬念 002，防止过期
```

### 每章规划悬念

在写新章前，告诉 AI：

```text
本章我要推进悬念 001（给出新线索），
同时提及悬念 002（防止过期）。
```

AI 会自动在写作中实现。

## 常见问题

### Q: 我可以不写大纲直接开始吗？

可以。对 AI 说：
```text
直接开始写，不写大纲。
```

AI 会生成一页「精简总纲」后开始写第 1 章。

但推荐先写大纲，因为：
- 写到第 10 章不会崩盘
- 结局不会烂尾
- 悬念不会忘

### Q: AI 写的我不满意怎么办？

**修改某一段**：
```text
修改第 X 章第 Y 段，问题是[具体问题]。
例如：对白不自然 / 节奏太慢 / 缺少细节
```

**重写整章**：
```text
重写第 X 章，要求：[具体要求]。
例如：钩子更强 / 对白更口语化 / 增加动作描写
```

**切换快速/标准模式**：
```text
快速写第 X 章（跳过场景拆分）
```

### Q: 我可以用自己的大纲吗？

可以。把你的大纲保存为 `00-大纲.md`，然后说：
```text
使用我的大纲，继续写第 X 章。
```

### Q: 可以多人协作吗？

可以。使用 `08-人机协作-v2.md` 的交接协议：

```markdown
## 人类修改记录
- 修改位置：第 X 章 第 Y 段
- 修改原因：___
- 继续要求：___
```

保存后，另一位作者可以继续。

## 最后提醒

1. **第一遍写**，不要追求完美
   - 用快速模式
   - 先完成，再完美

2. **第二遍改**，重点去 AI 味
   - 运行 `check_ai_style.py`
   - 对照 `ai-style-examples.md` 改写

3. **第三遍润**，打磨节奏和钩子
   - 对照 `hook-techniques.md`
   - 检查每章结尾钩子

**三遍之后，你的小说已经比 90% 的 AI 生成内容更像人类写的。**
